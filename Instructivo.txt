GIT:

git pull
Descarga los cambios del repositorio remoto y los fusiona con tu rama local actual.

git add .
Agrega todos los archivos modificados o nuevos al área de staging, listos para hacer commit.

git commit -m "mensaje"
Guarda en el repositorio local los cambios preparados, con un mensaje descriptivo.

git push
Envía los commits locales al repositorio remoto (por ejemplo, GitHub).

git diff
Muestra las diferencias entre los archivos modificados y el último commit.

git status
Muestra el estado actual del repositorio: qué archivos cambiaron y cuáles están en staging.

git reset
Deshace cambios. Puede usarse para quitar archivos del staging o volver a un commit anterior (con cuidado).
